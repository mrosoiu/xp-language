RFC 0088: Streams API

@category: io

Scope of Change
---------------
A new streams API will be created.


Rationale
---------
Add generic, reusable stream interfaces.


Functionality
-------------

Basics
~~~~~~
Two new interfaces will be created (pseudo-code follows):
<?php
  interface io.InputStream {
    string read(int nbytes= 8192) throws io.IOException
    void close() throws io.IOException
  }

  interface io.OutputStream {
    int write(string data) throws io.IOException
    void close() throws io.IOException
  }
?>

The io.Stream class will implement both of these interfaces.


Changes: Imaging API
~~~~~~~~~~~~~~~~~~~~
* The img.io.StreamWriter class will accept an io.OutputStream
* The img.io.StreamReader class will accept an io.InputStream


Changes: Text API
~~~~~~~~~~~~~~~~~
* The text.CSVGenerator class will accept an io.OutputStream
* The text.parser.CSVParser class will accept an io.InputStream


Changes: Remote API
~~~~~~~~~~~~~~~~~~~
* The remote.protocol.ByteCountedString class' writeTo() method
  will accept an io.OutputStream
* The remote.protocol.ByteCountedString class' readRrom() method
  will accept an io.InputStream


Changes: Peer API
~~~~~~~~~~~~~~~~~
* The peer.ldap.util.LDIFWriter class will accept an io.OutputStream


Enhancement: Scriptlet API
~~~~~~~~~~~~~~~~~~~~~~~~~~
* The HttpScriptletResponse class will implement the io.OutputStream 
  interface
  

Enhancement: I/O Collections API
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
* The io.collections.IOElement interface will receive two new methods:
  asInputStream()
  asOutputStream()
  to support modifying the elements.


Security considerations
-----------------------
n/a


Speed impact
------------
Slightly slower.


Dependencies
------------
None. All changes are BC-break free.
